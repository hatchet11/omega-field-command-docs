Evidence, not adjectives

Every vendor says secure. We prove it.

These are the claims we can hand a procurement office in writing, and back with a test.

Agencies cannot see each other's data

Isolation is enforced by PostgreSQL row-level security, not an application WHERE clause, so the database refuses even when the application is wrong. An automated test applies the real migrations to a live database and asserts a cross-agency read returns nothing. The build fails if that proof does not run.

No other vendor's product code is inside

The web framework, authentication, mapping, and geocoder are Omega Point original work on United States foundations. The production build adds exactly two registered dependencies, and a build gate fails on anything else.

Officer safety is a correctness property

Staleness, sharing state, and delivery are enforced invariants, not display logic. The suite asserts the denials first: cross-agency access, stale shown as live, a supervisor force-enabling sharing, and a replayed one-time code.

No certification is claimed

Controls are described as designed toward CJIS and related standards. We do not assert a certification until an independent assessor validates it. We would rather lose a checkbox than put a claim in front of an agency we cannot substantiate.

Capabilities

Built for the field, and for the compliance office.

Location that never lies

Stale is labeled stale, GPS off renders off. One render path, so nothing can show an old fix as current.

One-tap assistance

Per-recipient delivery state, acknowledgement, and a mandatory disposition on close. Delivered means a transport ACK, never assumed.

Structural tenant isolation

Per-agency row-level security at the database, forced for the owner, proven against a live PostgreSQL on every commit.

Accountability by construction

Append-only, hash-chained audit of every action including the denials. Coordinates and content scrubbed before write.

American made, enforced

Two United States origin, permissively licensed dependencies. A CI gate parses every module and rejects anything else.

In-house mapping

Web Mercator, a Census-method geocoder, and offline packages. Agencies can connect existing GIS, but none is required.

Shape the pilot.

Phase 2 build. Identity, authorization, location, assistance, audit, and mapping are implemented and tested. Not yet released for production caseload, and no live agency data is in use. Early agencies shape the roadmap.
